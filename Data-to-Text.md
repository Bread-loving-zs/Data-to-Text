# Data\-to\-Text

### 核心架构：Agent \+ LLM \+ 外部工具

由于报告中涉及数据检验（如卡方检验 $\chi^2=12.68, P<0.005$），**需要**调度数据库和代码工具来完成计算。

### 第一阶段：基座大模型的选择与私有化部署

- **基座模型**：Qwen3\-14B\-Base

- **参数量选择**：14B 参数量的模型可以通过 Ollama 等工具在单张消费级显卡上运行，降低前期的调试门槛。

- **部署方式**：使用Ollama 在本地服务器上进行私有化部署，确保数据不出域。

2. 将历史积累的“数据表 \+ 最终生成的报告段落”构造成问答对（JSONL格式）。

    - **Input \(输入\)**：结构化的抽检核心数据标签（例如：`{"年份":"xx年一季度", "抽检批次":45, "不合格批次":9, "上年不合格率":"14.89%", "P值":"<0.005"}`）。

    - **Output \(输出\)**：期望模型写出来的段落。

3. **微调技术**：使用 **LoRA** 技术进行微调，大概需要几百到上千条高质量的“数据\-文本”对照样本。微调后的模型在看到类似的数据指标时，就能模仿标准文风写出流畅的分析段落。

### 第三阶段：构建工作流智能体

1. **意图识别**：当用户在聊天窗口输入“帮我生成一份2024年一季度的辣椒抽检风险报告”时，Agent 能够识别关键实体（时间：2024一季度，品种：辣椒）。

2. **调用数据库工具 \(Text\-to\-SQL / API\)**：Agent 自动将提取的实体转化为 SQL 查询语句，或者调用你们预先写好的后台接口，从数据库中拉取相应的统计数据。

3. **调用计算工具 \(Python Code Interpreter\)**：对于复杂的统计学意义分析（如卡方检验）或同比环比计算，Agent 应调用后台写好的 Python 数据分析脚本处理提取出的数据，得出确切结论。

4. **上下文组装**：Agent 将计算好的准确数据作为“参考资料（Context）”喂给微调过的大模型。

5. **文本生成**：大模型根据输入的数据和参考资料，自主组织语言生成报告文本。  

### 第四阶段：Word 版本报告生成工程

·大模型输出的内容通常是 Markdown 格式的纯文本，需要将其转化为Word 文档。

- **使用库**：在 Python 中使用 `python-docx` 库。

- **图表插入**：文档中提到报告里包含“图1”、“图2”。通过 Python 的 `matplotlib` 或 `pyecharts` 库，利用步骤三中查出的数据自动生成统计图表（如折线图、饼图），然后通过 `python-docx` 将生成的图表图片插入到 Word 文档的指定位置中。  

- **模板替换**：提供一个带有特定样式的 Word 空白模板，用代码将大模型生成的各章节文本精确地写入对应的标题级别下。

通过这四个阶段，能实现从“填空式的模板套用”升级为“AI智能查数、智能分析、智能撰写”的完整闭环。
